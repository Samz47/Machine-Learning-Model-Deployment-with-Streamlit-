## üìä Dataset

### Wine Quality Dataset

**Source**: [Kaggle - Wine Quality Dataset](https://www.kaggle.com/datasets/yasserh/wine-quality-dataset)

**Description**: The dataset contains various physicochemical properties of wines and their quality ratings.

**Features**:
- `fixed acidity`: Fixed acidity content
- `volatile acidity`: Volatile acidity content
- `citric acid`: Citric acid content
- `residual sugar`: Residual sugar content
- `chlorides`: Chloride content
- `free sulfur dioxide`: Free sulfur dioxide content
- `total sulfur dioxide`: Total sulfur dioxide content
- `density`: Wine density
- `pH`: pH level
- `sulphates`: Sulphate content
- `alcohol`: Alcohol content
- `quality`: Quality score (target variable, 0-10)

### Download Instructions

1. Visit the [Kaggle dataset page](https://www.kaggle.com/datasets/yasserh/wine-quality-dataset)
2. Download the dataset (CSV file)
3. Place the CSV file in the `data/` directory as `winequality.csv`
4. Alternatively, you can use the UCI Wine Quality dataset which is publicly available

**Note**: If you don't have the dataset, the app will still work but some features may be limited. The model files (`model.pkl`, etc.) should be generated by running the training notebook first.

## üöÄ Installation

### Prerequisites

- Python 3.8 or higher
- pip (Python package manager)

### Step 1: Clone the Repository

```bash
git clone https://github.com/Samz47/Machine-Learning-Model-Deployment-with-Streamlit-.git
cd wine-quality-prediction
```

### Step 2: Create Virtual Environment (Recommended)

```bash
# Windows
python -m venv venv
venv\Scripts\activate

# macOS/Linux
python3 -m venv venv
source venv/bin/activate
```

### Step 3: Install Dependencies

```bash
pip install -r requirements.txt
```

### Step 4: Download Dataset

1. Download the wine quality dataset from [Kaggle](https://www.kaggle.com/datasets/yasserh/wine-quality-dataset)
2. Place the CSV file in the `data/` directory as `winequality.csv`

### Step 5: Train the Model

1. Open `notebooks/model_training.ipynb` in Jupyter Notebook or JupyterLab
2. Run all cells to:
   - Load and explore the data
   - Preprocess the data
   - Train multiple models
   - Evaluate and compare models
   - Save the best model and related files

**Note**: The model files will be saved in the root directory after running the notebook.

## üî¨ Model Training

### Models Trained

1. **Random Forest Classifier** (Best Model)
   - Handles non-linear relationships
   - Provides feature importance
   - Good performance on this dataset

2. **Logistic Regression**
   - Linear model with regularization
   - Fast training and prediction
   - Requires feature scaling

3. **Support Vector Machine (SVM)**
   - Effective for classification
   - Requires feature scaling
   - Can handle non-linear relationships with kernels

### Model Evaluation

- **Cross-Validation**: 5-fold cross-validation for robust evaluation
- **Metrics**: Accuracy, Precision, Recall, F1-Score
- **Comparison**: Side-by-side comparison of all models

### Model Selection

The Random Forest Classifier was selected as the best model based on:
- Highest test accuracy
- Best cross-validation performance
- Feature importance interpretability
- Robustness to overfitting

## üåê Deployment

### Deploying to Streamlit Cloud

1. **Push to GitHub**
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin <your-github-repo-url>
   git push -u origin main
   ```

2. **Deploy on Streamlit Cloud**
   - Go to [share.streamlit.io](https://share.streamlit.io)
   - Sign in with your GitHub account
   - Click "New app"
   - Select your repository
   - Set the main file path to `app.py`
   - Click "Deploy"

3. **Configure Deployment**
   - Ensure `requirements.txt` is in the root directory
   - All model files (`.pkl`) should be committed or generated during deployment
   - If model files are large (>100MB), consider using Git LFS

### Important Notes for Deployment

- **Model Files**: Ensure all `.pkl` files are committed to the repository
- **Dataset**: The dataset should be in the `data/` directory
- **Dependencies**: All dependencies must be listed in `requirements.txt`
- **File Paths**: Use relative paths (not absolute paths) in the code
